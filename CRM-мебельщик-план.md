# План: простая CRM для мебельщика (нержавеющая сталь)

Онлайн-система с доступом с телефона. Ниже — план и готовый промпт для Claude Code (Opus) в VS Code.

---

## 1. Что получится

- Веб-приложение, открывается с компьютера и телефона (адаптивный интерфейс + PWA — можно добавить на главный экран телефона как приложение).
- Вход по паролю (один пользователь — сам мебельщик).
- Клиенты: имя, телефон, адрес, заметки.
- Заказы: клиент, описание изделия, статус (Заявка → Замер → Производство → Монтаж → Завершён), сумма, предоплата, остаток (считается сам), срок сдачи, заметки.
- Напоминания: раздел «Сегодня» на главной — замеры и сдачи на сегодня/просроченные, заказы с неоплаченным остатком.
- Поиск по клиентам и заказам.

## 2. Технологии (просто и бесплатно)

| Часть | Выбор | Почему |
|---|---|---|
| Приложение | Next.js 14 + TypeScript + Tailwind | Один проект = и интерфейс, и сервер |
| База данных | PostgreSQL на Neon.tech (бесплатный тариф) | Ничего не надо администрировать |
| ORM | Prisma | Простая работа с базой |
| Хостинг | Vercel (бесплатный тариф) | Деплой одной командой, HTTPS из коробки |
| Вход | Простой пароль в переменной окружения + cookie-сессия | Без сложной регистрации |

## 3. Порядок работы в VS Code

1. Установить Node.js LTS и Claude Code.
2. Создать пустую папку проекта, открыть в VS Code, запустить `claude`.
3. Вставить промпт из раздела 5 целиком.
4. Проверить локально: `npm run dev` → http://localhost:3000.
5. Зарегистрироваться на neon.tech, создать базу, скопировать строку подключения в `.env`.
6. Задеплоить: аккаунт на vercel.com → `npx vercel` → добавить переменные окружения (DATABASE_URL, APP_PASSWORD) в настройках Vercel.
7. Открыть ссылку с телефона → «Добавить на главный экран».

## 4. Модель данных

```
Client:  id, name, phone, address, notes, createdAt
Order:   id, clientId, title, description, status, price, prepaid,
         measureDate (дата замера), dueDate (срок сдачи),
         notes, createdAt, updatedAt
Status:  NEW | MEASURE | PRODUCTION | INSTALL | DONE
```

## 5. Готовый промпт для Claude Code (Opus) — вставить целиком

```
Создай простую CRM-систему для мастера, который делает мебель из нержавеющей стали
(перила, столы, стеллажи, кухонные конструкции). Один пользователь. Интерфейс на русском.

СТЕК:
- Next.js 14 (App Router) + TypeScript + Tailwind CSS
- Prisma + PostgreSQL (строка подключения из env DATABASE_URL; для локальной
  разработки настрой docker-compose с postgres ИЛИ подключи Neon)
- Деплой на Vercel. Добавь manifest.json и иконки, чтобы работало как PWA на телефоне.

АВТОРИЗАЦИЯ:
- Страница входа с одним полем «Пароль». Пароль сравнивается с env APP_PASSWORD.
- При успехе — httpOnly cookie-сессия на 30 дней. Все остальные страницы защищены middleware.

ДАННЫЕ (Prisma schema):
- Client: id, name (обязательное), phone, address, notes, createdAt
- Order: id, clientId (связь с Client), title (обязательное), description,
  status enum: NEW («Заявка»), MEASURE («Замер»), PRODUCTION («Производство»),
  INSTALL («Монтаж»), DONE («Завершён»),
  price Decimal, prepaid Decimal (default 0), measureDate DateTime?, dueDate DateTime?,
  notes, createdAt, updatedAt
- Остаток к оплате = price - prepaid, вычисляется, в базе не хранится.

СТРАНИЦЫ:
1. /login — вход.
2. / — главная «Сегодня»: три блока-напоминания:
   - «Замеры»: заказы с measureDate сегодня или просроченной (не DONE), красным если просрочено;
   - «Сроки сдачи»: заказы с dueDate в ближайшие 3 дня или просроченной (не DONE);
   - «Ждут оплаты»: заказы DONE с остатком > 0.
   Плюс счётчики: заказов в работе, сумма остатков к оплате.
3. /orders — список заказов: карточки с названием, клиентом, статусом (цветной бейдж),
   суммой/остатком, сроком. Фильтр по статусу, поиск по названию и имени клиента,
   кнопка «+ Новый заказ».
4. /orders/[id] — карточка заказа: все поля, смена статуса одним нажатием
   (кнопки-шаги воронки), редактирование, удаление с подтверждением.
5. /clients — список клиентов с поиском, «+ Новый клиент».
6. /clients/[id] — карточка клиента: данные, кнопка позвонить (tel: ссылка),
   список его заказов, редактирование, удаление.

ИНТЕРФЕЙС:
- Mobile-first: удобно с телефона, крупные кнопки, нижняя панель навигации
  (Сегодня / Заказы / Клиенты) на мобильном, обычное меню на десктопе.
- Простой чистый дизайн, без лишнего. Цвета статусов: серый/синий/оранжевый/фиолетовый/зелёный.
- Суммы в формате «12 500 ₽», даты «15 июля 2026».

КАЧЕСТВО:
- Server Actions или API routes с валидацией (zod).
- Seed-скрипт с 3 тестовыми клиентами и 5 заказами в разных статусах.
- README.md на русском: как запустить локально, как создать базу на Neon,
  как задеплоить на Vercel, какие переменные окружения нужны.
- .env.example с DATABASE_URL и APP_PASSWORD.

Сделай проект полностью рабочим: prisma migrate, seed, npm run dev без ошибок.
В конце проверь сборку командой npm run build.
```

# CRM Мебель

Простая CRM для мастера, изготавливающего мебель из нержавеющей стали (перила,
столы, стеллажи, кухонные конструкции). Один пользователь, вход по паролю.
Работает с телефона как PWA.

## Возможности

- **Сегодня** — напоминания: замеры на сегодня/просроченные, ближайшие сроки сдачи,
  заказы, ожидающие оплаты; счётчики заказов в работе и суммы к оплате.
- **Заказы** — список с поиском и фильтром по статусу, воронка статусов
  (Заявка → Замер → Производство → Монтаж → Завершён), сумма/предоплата/остаток.
- **Клиенты** — карточки с телефоном (кнопка «Позвонить»), адресом и списком заказов.
- Адаптивный интерфейс (нижняя навигация на телефоне), тёмная/светлая тема
  автоматически.

## Технологии

- Next.js 14 (App Router) + TypeScript + Tailwind CSS
- Prisma + PostgreSQL
- Server Actions с валидацией zod
- Авторизация: пароль + подписанная httpOnly cookie-сессия (30 дней)

## Переменные окружения

| Переменная       | Назначение                                                |
| ---------------- | --------------------------------------------------------- |
| `DATABASE_URL`   | Строка подключения к PostgreSQL                           |
| `APP_PASSWORD`   | Пароль для входа в систему                                 |
| `SESSION_SECRET` | Секрет для подписи cookie-сессии (минимум 16–32 символа)  |

Шаблон — в файле [.env.example](.env.example). Скопируйте его в `.env` и заполните.

## Запуск локально

Требуется **Node.js 18+** и **Docker** (для локальной базы).

```bash
# 1. Зависимости
npm install

# 2. Локальная база PostgreSQL в Docker (порт 5434)
npm run db:up

# 3. Файл окружения
cp .env.example .env      # значения по умолчанию уже подходят для локального Docker

# 4. Применить схему БД
npm run db:migrate

# 5. Тестовые данные (3 клиента, 5 заказов)
npm run db:seed

# 6. Запуск в режиме разработки
npm run dev
```

Откройте <http://localhost:3000>. Пароль по умолчанию — `changeme` (из `.env`).

### Полезные команды

```bash
npm run test        # unit-тесты (vitest)
```

## База данных на Neon (для продакшена)

[Neon](https://neon.tech) — бесплатный облачный PostgreSQL.

1. Зарегистрируйтесь на <https://neon.tech> и создайте проект (Project).
2. Скопируйте строку подключения (Connection string) вида
   `postgresql://user:password@ep-xxx.eu-central-1.aws.neon.tech/neondb?sslmode=require`.
3. Вставьте её в `DATABASE_URL` (локально в `.env` или в переменные окружения Vercel).
4. Примените схему к облачной базе:

   ```bash
   npx prisma migrate deploy
   npm run db:seed        # по желанию — тестовые данные
   ```

## Деплой на Vercel

1. Залейте проект в репозиторий GitHub/GitLab.
2. На <https://vercel.com> нажмите **Add New → Project** и импортируйте репозиторий.
3. В настройках проекта добавьте переменные окружения:
   - `DATABASE_URL` — строка подключения к Neon,
   - `APP_PASSWORD` — ваш пароль для входа,
   - `SESSION_SECRET` — случайная строка (например, из `openssl rand -hex 32`).
4. Нажмите **Deploy**. Команда сборки (`npm run build`) уже включает
   `prisma generate`.
5. После первого деплоя примените миграции к базе Neon (однократно, локально):

   ```bash
   npx prisma migrate deploy
   ```

6. Откройте выданный адрес на телефоне и добавьте на главный экран
   («Поделиться → На экран Домой» или «⋮ → Установить приложение»).

## Структура

```
prisma/                — схема, миграции, seed
scripts/gen-icons.mjs  — генерация иконок PWA
src/
  actions/             — Server Actions (auth, clients, orders)
  app/                 — страницы (App Router)
  components/          — UI-компоненты
  lib/                 — prisma, auth, валидация, форматирование, напоминания, статусы
middleware.ts          — защита маршрутов
docker-compose.yml     — локальный PostgreSQL
```

## Смена валюты

По умолчанию суммы отображаются в тенге (`12 500 ₸`). Чтобы изменить символ или
локаль, отредактируйте `formatMoney` в [src/lib/format.ts](src/lib/format.ts).

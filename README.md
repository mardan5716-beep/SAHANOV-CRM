# Glint & Steel CRM

CRM для серийного производства и продажи аксессуаров из нержавеющей стали (полки,
консоли, приставные столики, подставки, этажерки, органайзеры, зеркала и т.д.).
Работает с телефона как PWA. Принцип: менеджер оформляет сделку меньше чем за минуту.

## Возможности

- **Дашборд** — KPI: активные сделки, сумма к оплате, выручка и маржа за месяц;
  список товаров с низким остатком; последние сделки.
- **Сделки** — позиции подбираются из каталога склада (артикул подтягивает цену и
  себестоимость), скидки в % или ₸, живой пересчёт суммы и маржи. Воронка из 10
  статусов, оплата (Kaspi/наличные/карта/перевод) и доставка (самовывоз/доставка,
  трек-номер). Номер заказа генерируется автоматически (`GS-00001`).
- **Склад** — товары с артикулом, категорией, ценой, себестоимостью, остатком и
  минимальным остатком; подсветка низких остатков. **Автосписание**: при переводе
  сделки в «Отправлено» остатки уменьшаются автоматически, при «Возврат»/«Отмена» —
  возвращаются.
- **Клиенты** — имя, телефон, компания, источник; история сделок, звонок по ссылке.
- **Менеджеры** — пользователи с входом по email; менеджер сделки = вошедший.
- Мягкое удаление (данные не стираются физически, их можно восстановить),
  адаптивный интерфейс, тёмная/светлая тема, суммы в тенге (₸).

## Вход и роли

Вход по **email + паролю**. Пароли хранятся хешированными (PBKDF2). Первый админ
создаётся через seed: **`admin@glint.kz` / `changeme`** (смените после первого входа).

- **Админ** — управляет складом (товары, цены, себестоимость) и менеджерами,
  видит себестоимость и маржу, может выдать менеджеру админ-права.
- **Менеджер** — оформляет сделки и клиентов, видит каталог и остатки, суммы и
  оплату; себестоимость и маржа скрыты.

Новых пользователей добавляет админ на странице «Менеджеры».

## Технологии

- Next.js 14 (App Router) + TypeScript + Tailwind CSS
- Prisma + PostgreSQL
- Server Actions с валидацией zod
- Авторизация: email + пароль (PBKDF2), подписанная httpOnly cookie-сессия (30 дней)

## Модель данных

`Manager` · `Client` · `Product` (склад) · `Order` (сделка, содержит оплату и
доставку) · `OrderItem` (позиция со снимком артикула/цены/себестоимости на момент
заказа). Суммы, остаток к оплате и маржа считаются в коде и не хранятся в базе.

## Переменные окружения

| Переменная       | Назначение                                               |
| ---------------- | -------------------------------------------------------- |
| `DATABASE_URL`   | Строка подключения к PostgreSQL                          |
| `SESSION_SECRET` | Секрет для подписи cookie-сессии (минимум 16–32 символа) |

Шаблон — в [.env.example](.env.example).

## Запуск локально

Требуется **Node.js 18+** и **Docker**.

```bash
npm install            # зависимости
npm run db:up          # PostgreSQL в Docker (порт 5434)
cp .env.example .env    # значения по умолчанию подходят для локального Docker
npm run db:migrate     # применить схему
npm run db:seed        # тестовые данные (менеджеры, клиенты, товары, сделки)
npm run dev            # http://localhost:3000
```

Вход по умолчанию (из seed): **`admin@glint.kz` / `changeme`**.

### Команды

```bash
npm run test        # unit-тесты (расчёты, нумерация, валидация, форматирование, сессия)
npm run build       # production-сборка
npm run db:reset    # пересоздать базу и применить миграции заново
npm run db:down     # остановить контейнер с базой
```

## База данных на Neon (продакшен)

1. Зарегистрируйтесь на <https://neon.tech>, создайте проект.
2. Скопируйте строку подключения в `DATABASE_URL`.
3. Примените схему: `npx prisma migrate deploy`, при желании `npm run db:seed`.

## Деплой на Vercel

1. Залейте проект в Git-репозиторий.
2. На <https://vercel.com> импортируйте репозиторий (**Add New → Project**).
3. Добавьте переменные окружения: `DATABASE_URL` (Neon), `SESSION_SECRET`
   (случайная строка, `openssl rand -hex 32`).
4. **Deploy** (сборка уже включает `prisma generate`).
5. Однократно примените миграции и создайте админа: `npx prisma migrate deploy`,
   затем `npm run db:seed` (или добавьте менеджера-админа вручную) и **смените пароль**.
6. Откройте адрес на телефоне → «Добавить на главный экран».

## Смена валюты

Суммы отображаются в тенге (`12 500 ₸`). Символ/локаль — в `formatMoney`
([src/lib/format.ts](src/lib/format.ts)).

## Структура

```
prisma/                 — схема, миграции, seed
src/
  actions/              — Server Actions (auth, clients, products, managers, orders)
  app/                  — страницы (App Router)
  components/           — UI-компоненты
  lib/                  — prisma, auth, валидация, форматирование, enums,
                          order-calc (суммы/маржа), order-number, dashboard
src/middleware.ts       — защита маршрутов
docker-compose.yml      — локальный PostgreSQL
```
